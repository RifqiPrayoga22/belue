Jenis-jenis Constructor di Dart

Theory — Konstruktor

Constructor adalah method khusus dalam sebuah class yang digunakan untuk membuat objek. Constructor dipanggil otomatis saat objek dibuat dan berfungsi menginisialisasi nilai awal dari properti class.

Default Constructor

Dibuat otomatis jika tidak ada constructor lain yang didefinisikan.

Parameterized Constructor

Menerima parameter untuk inisialisasi nilai field saat objek dibuat.

Named Constructor

Constructor dengan nama khusus: Class.namaConstructor()

Redirecting Constructor

Meneruskan pembuatan objek ke constructor lain dalam class yang sama.

dart
class Mahasiswa {
  String nama = '';
  int umur = 0;
  Mahasiswa() { nama = 'Tidak Diketahui'; umur = 0; }
  void tampilkan() { print('Nama: $nama | Umur: $umur'); }
}
class MahasiswaParam {
  String nama; int umur;
  MahasiswaParam(String nama, int umur) { this.nama = nama; this.umur = umur; }
  MahasiswaParam.shorthand(this.nama, this.umur);
  void tampilkan() { print('Nama: $nama | Umur: $umur'); }
}
void main() {
  var m1 = Mahasiswa(); m1.tampilkan();
  var m2 = MahasiswaParam('Rifqi', 21); m2.tampilkan();
  var m3 = MahasiswaParam.shorthand('Prayoga', 22); m3.tampilkan();
}
Output ✓ PASSED
▶Nama: Tidak Diketahui | Umur: 0
▶Nama: Rifqi | Umur: 21
▶Nama: Prayoga | Umur: 22

Constant Constructor

Membuat objek yang tidak dapat diubah (immutable)

Constant constructor digunakan untuk membuat objek yang bersifat immutable. Ditandai dengan keyword const dan semua field harus bertipe final.

Syarat Utama

Semua field dalam class harus bertipe final.

Hemat Memori

Objek dengan nilai yang sama hanya dibuat sekali — identik di memori.

Compile-time Constant

Nilai sudah diketahui saat kompilasi, lebih efisien.

💡Gunakan identical(t1, t2) untuk membuktikan bahwa dua const object dengan nilai sama adalah objek yang benar-benar sama di memori.
dart
class TitikKoordinat {
  final double x; final double y;
  const TitikKoordinat(this.x, this.y);
}
class WarnaRGB {
  final int merah; final int hijau; final int biru;
  const WarnaRGB(this.merah, this.hijau, this.biru);
  const WarnaRGB.merahPrimary() : merah = 255, hijau = 0, biru = 0;
  const WarnaRGB.hijauPrimary() : merah = 0, hijau = 255, biru = 0;
}
void main() {
  const t1 = TitikKoordinat(3.0, 4.0);
  const t2 = TitikKoordinat(3.0, 4.0);
  print(identical(t1, t2)); // true
  const merah = WarnaRGB.merahPrimary();
  print(merah.merah);
  print('RGB: ${merah.merah}, ${merah.hijau}, ${merah.biru}');
}
Output ✓ PASSED
▶true
▶255
▶RGB: 255, 0, 0

Constructor dengan Initializer List

Inisialisasi field sebelum body constructor dijalankan

Initializer list menginisialisasi nilai variabel sebelum body constructor dijalankan. Ditulis menggunakan tanda titik dua (:) setelah parameter constructor.

⚡Initializer list berjalan SEBELUM body constructor. Ini satu-satunya cara untuk menginisialisasi field final yang nilainya perlu dihitung dari parameter.
dart
class Segitiga {
  final double alas; final double tinggi; final double luas;
  Segitiga(this.alas, this.tinggi)
      : luas = 0.5 * alas * tinggi {
    print('Segitiga dibuat: alas=$alas, tinggi=$tinggi, luas=$luas');
  }
}
class Lingkaran {
  final double jariJari; final double keliling; final double luasLingkaran;
  Lingkaran(this.jariJari)
      : keliling = 2 * 3.14159 * jariJari,
        luasLingkaran = 3.14159 * jariJari * jariJari;
}
void main() {
  var s = Segitiga(6.0, 4.0); print(s.luas);
  var l = Lingkaran(7.0); print(l.keliling); print(l.luasLingkaran);
}
Output ✓ PASSED
▶Segitiga dibuat: alas=6.0, tinggi=4.0, luas=12.0

Pengantar Enkapsulasi

Menyembunyikan detail internal dari dunia luar

Enkapsulasi adalah konsep OOP yang menyembunyikan detail implementasi internal sebuah class dan hanya mengekspos bagian yang diperlukan. Tujuannya adalah melindungi data dari perubahan yang tidak sah.

🛡 Keamanan Data

Data tidak bisa diakses atau diubah sembarangan dari luar class.

🌓 Kontrol Akses

Hanya method yang diizinkan (getter/setter) yang bisa mengubah data.

🔧 Fleksibilitas

Implementasi internal bisa diubah tanpa merusak kode yang menggunakan class.

✅ Validasi

Data dapat divalidasi terlebih dahulu sebelum disimpan melalui setter.

⚡Dart tidak punya keyword private seperti Java/C++. Sebagai gantinya, Dart menggunakan underscore (_) di depan nama field/method untuk menandainya sebagai private.

Public vs Private di Dart

Mengontrol akses terhadap data

Dart tidak memiliki keyword private atau public. Akses dikontrol menggunakan prefix underscore (_) yang membuat member hanya dapat diakses di dalam file/library yang sama.

✅ PUBLIC

  • Tidak ada prefix khusus
  • Dapat diakses dari mana saja
  • Dari class lain, file lain, library lain
  • Contoh: nama, umur
  • Contoh method: tampilkan()

🔒 PRIVATE

  • Prefix underscore _ sebelum nama
  • Hanya bisa diakses dalam file yang sama
  • Tidak bisa diakses dari file lain
  • Contoh: _nama, _password
  • Contoh method: _hitungInternal()
dart
class Contoh {
  String namaPubik = 'visible';   // PUBLIC  — bisa diakses dari luar
  String _namaPrivate = 'hidden'; // PRIVATE — prefix _
  int _umur = 20;
  void metodePubik() { print('Metode publik dipanggil'); }
  void _metodePrivate() { print('Metode private dipanggil'); }
}
void main() {
  var c = Contoh();
  print(c.namaPubik);   // ✓ OK
  c.metodePubik();     // ✓ OK
  // print(c._namaPrivate); ❌ ERROR jika di file berbeda
  // c._metodePrivate();    ❌ ERROR jika di file berbeda
}
Output ✓ PASSED
▶visible
▶Metode publik dipanggil

Getter — Mengakses Data Private

Mengizinkan pembacaan field private secara terkontrol

Getter adalah method khusus dengan keyword get untuk membaca nilai field private. Getter boleh berisi logika komputasi — tidak hanya mengembalikan field secara langsung.

Sintaks: TipeData get namaProperty => _field; Boleh berisi logika Tidak ada parameter
dart
class Mahasiswa {
  String _nama; int _nim; double _nilaiAkhir;
  Mahasiswa(this._nama, this._nim, this._nilaiAkhir);
  String get nama => _nama;
  int get nim => _nim;
  String get grade {
    if (_nilaiAkhir >= 90) return 'A';
    if (_nilaiAkhir >= 80) return 'B';
    if (_nilaiAkhir >= 70) return 'C';
    return 'D';
  }
  String get statusKelulusan => _nilaiAkhir >= 60 ? 'LULUS' : 'TIDAK LULUS';
  String get nimFormatted => 'NIM: ${_nim.toString().padLeft(8, "0")}';
}
void main() {
  var m = Mahasiswa('Rifqi', 251410005, 87.5);
  print(m.nama); print(m.grade);
  print(m.statusKelulusan); print(m.nimFormatted);
}
Output ✓ PASSED
▶Rifqi
▶B
▶LULUS
▶NIM: 251410005

Setter — Memodifikasi Data Private

Mengizinkan perubahan data dengan validasi

Setter menggunakan keyword set dan memungkinkan validasi data sebelum disimpan. Setter tidak memiliki return value. Gunakan throw Exception() untuk menolak nilai yang tidak valid.

Sintaks: set namaProperty(TipeData val) { ... } Tidak ada return value Validasi sebelum simpan
dart
class User {
  String _username; String _password; String _email;
  User(this._username, this._password, this._email);
  String get username => _username;
  String get email => _email;
  set username(String val) {
    if (val.length < 5) throw Exception('Username minimal 5 karakter!');
    _username = val;
  }
  set password(String val) {
    if (val.length < 8) throw Exception('Password minimal 8 karakter!');
    _password = val;
  }
  set email(String val) {
    if (!val.contains('@')) throw Exception('Email harus mengandung "@"!');
    _email = val;
  }
}
void main() {
  var user = User('rifqi1', 'password1', '[email protected]');
  user.username = 'prayoga'; user.email = '[email protected]';
  try { user.username = 'rif'; } catch (e) { print(e); }
  try { user.email = 'tidakvalid'; } catch (e) { print(e); }
}
Output ✓ PASSED
▶Exception: Username minimal 5 karakter!
▶Exception: Email harus mengandung "@"!

Multiple Constructor (Latihan)

Named constructor — berbagai cara membuat objek

Multiple constructor memungkinkan sebuah class memiliki lebih dari satu constructor. Di Dart, ini dilakukan dengan named constructor. Setiap constructor bisa memiliki logika inisialisasi yang berbeda.

dart
class Mahasiswa {
  String nama; int nim; double ipk;
  Mahasiswa() : nama = 'Tidak Diketahui', nim = 0, ipk = 0.0;
  Mahasiswa.dariMap(Map<String, dynamic> data)
      : nama = data['nama'] ?? 'Tidak Diketahui',
        nim  = data['nim']  ?? 0,
        ipk  = (data['ipk'] ?? 0.0).toDouble();
  Mahasiswa.tanpaNama(this.nim, this.ipk) : nama = 'Anonim';
  void tampilkan() { print('Nama: $nama | NIM: $nim | IPK: $ipk'); }
}
void main() {
  var m1 = Mahasiswa();
  var m2 = Mahasiswa.dariMap({'nama': 'Rifqi', 'nim': 251410005, 'ipk': 3.75});
  var m3 = Mahasiswa.tanpaNama(67890, 3.20);
  m1.tampilkan(); m2.tampilkan(); m3.tampilkan();
}
Output ✓ PASSED
▶Nama: Tidak Diketahui | NIM: 0 | IPK: 0.0
▶Nama: Rifqi | NIM: 251410005 | IPK: 3.75
▶Nama: Anonim | NIM: 67890 | IPK: 3.2

Sistem Login Sederhana (Studi Kasus)

Enkapsulasi dalam skenario nyata

Contoh kasus ini menunjukkan bagaimana enkapsulasi diterapkan dalam sistem login. Data sensitif disimpan sebagai field private dan hanya bisa diakses melalui method yang telah ditentukan.

dart
class AkunPengguna {
  String _username; String _password;
  bool _isLoggedIn = false; int _loginAttempts = 0;
  static const int _maxAttempts = 3;
  AkunPengguna(this._username, this._password);
  String get username => _username;
  bool get isLoggedIn => _isLoggedIn;
  bool get isLocked => _loginAttempts >= _maxAttempts;
  String get status => _isLoggedIn ? 'Online' : 'Offline';
  set password(String newPass) {
    if (newPass.length < 8) throw Exception('Password minimal 8 karakter!');
    _password = newPass; print('Password berhasil diubah.');
  }
  bool login(String user, String pass) {
    if (isLocked) { print('Akun terkunci!'); return false; }
    if (_username == user && _password == pass) {
      _isLoggedIn = true; _loginAttempts = 0;
      print('Login berhasil! Selamat datang, $_username'); return true;
    }
    _loginAttempts++;
    print('Login gagal! Sisa percobaan: ${_maxAttempts - _loginAttempts}');
    return false;
  }
  void logout() { _isLoggedIn = false; print('Logout berhasil. Sampai jumpa, $_username!'); }
}
void main() {
  var akun = AkunPengguna('rifqi_dev', 'dart1234');
  print(akun.status);
  akun.login('rifqi_dev', 'salah');
  akun.login('rifqi_dev', 'dart1234');
  print(akun.status);
  akun.password = 'newpass99';
  akun.logout();
}
Output ✓ PASSED
▶Offline
▶Login gagal! Sisa percobaan: 2
▶Login berhasil! Selamat datang, rifqi_dev
▶Online
▶Password berhasil diubah.
▶Logout berhasil. Sampai jumpa, rifqi_dev!

🏆 Latihan — Simulasi Rekening Bank

Field private: _nomorRekening, _saldo  •  Method: deposit(), tarik(), cekSaldo()  •  Validasi: saldo tidak bisa negatif

Latihan ini menerapkan enkapsulasi pada class RekeningBank. Semua data rekening disimpan sebagai field private sehingga tidak bisa diakses atau diubah langsung dari luar class. Satu-satunya cara berinteraksi dengan data rekening adalah melalui method publik yang sudah dilengkapi validasi.

🔒 Field Private

_nomorRekening, _namaPemilik, _saldo — tidak bisa diakses langsung dari luar class.

⚙ Method Publik

deposit() menambah saldo • tarik() mengurangi saldo • cekSaldo() menampilkan info rekening.

✅ Validasi

Deposit > 0 • Tarik > 0 • Tarik ≤ saldo • Saldo tidak boleh negatif.

💡_saldo adalah field private — hanya bisa dibaca lewat getter saldo dan diubah lewat deposit() / tarik(). Validasi di dalam kedua method memastikan saldo tidak pernah turun ke bawah nol.
dart
class RekeningBank {
  // ── Field Private ───────────────────────────────────────────
  String _nomorRekening;
  String _namaPemilik;
  double _saldo;

  // ── Constructor ──────────────────────────────────────────
  RekeningBank(this._nomorRekening, this._namaPemilik, this._saldo);

  // ── Getter ──────────────────────────────────────────────
  String get nomorRekening => _nomorRekening;
  String get namaPemilik  => _namaPemilik;
  double get saldo         => _saldo;

  // ── Method deposit() ───────────────────────────────────────
  void deposit(double jumlah) {
    if (jumlah <= 0) {
      print('[ERROR] Jumlah deposit harus lebih dari 0!');
      return;
    }
    _saldo += jumlah;
    print('[DEPOSIT] +Rp${jumlah.toStringAsFixed(0)} ✅  Saldo: Rp${_saldo.toStringAsFixed(0)}');
  }

  // ── Method tarik() ──────────────────────────────────────────
  void tarik(double jumlah) {
    if (jumlah <= 0) {
      print('[ERROR] Jumlah penarikan harus lebih dari 0!');
      return;
    }
    if (jumlah > _saldo) {
      print('[ERROR] Saldo tidak cukup! Saldo saat ini: Rp${_saldo.toStringAsFixed(0)}');
      return;
    }
    _saldo -= jumlah;
    print('[TARIK]   -Rp${jumlah.toStringAsFixed(0)} ✅  Saldo: Rp${_saldo.toStringAsFixed(0)}');
  }

  // ── Method cekSaldo() ───────────────────────────────────────
  void cekSaldo() {
    print('================================');
    print('  INFO REKENING');
    print('================================');
    print('  No. Rekening : $_nomorRekening');
    print('  Pemilik      : $_namaPemilik');
    print('  Saldo        : Rp${_saldo.toStringAsFixed(0)}');
    print('================================');
  }
}

// ── Simulasi Transaksi ──────────────────────────────────────────
void main() {
  // Buat rekening baru dengan saldo awal Rp1.000.000
  var rek = RekeningBank('1234-5678-9012', 'Rifqi Prayoga', 1000000);
  rek.cekSaldo();

  // ── Transaksi valid ───────────────────────────────
  rek.deposit(500000);  // ✓ +500.000 → total 1.500.000
  rek.deposit(250000);  // ✓ +250.000 → total 1.750.000
  rek.tarik(200000);   // ✓ -200.000 → total 1.550.000

  // ── Transaksi tidak valid — validasi aktif ──────────
  rek.tarik(5000000);  // ✘ Saldo tidak cukup
  rek.deposit(-100);  // ✘ Jumlah tidak valid
  rek.tarik(0);        // ✘ Jumlah tidak valid

  rek.cekSaldo();      // Info akhir rekening
}

🔨 Praktik 2 — Enkapsulasi Lengkap

Class User: field private, getter semua field, setter dengan validasi username / password / email

Praktik ini menerapkan enkapsulasi secara lengkap pada class User. Semua data disimpan sebagai field private, dapat dibaca lewat getter, dan hanya bisa diubah lewat setter yang memiliki aturan validasi masing-masing. Setelah objek dibuat, kita coba modifikasi data dengan nilai valid dan tidak valid.

🔒 Field Private

_username, _password, _email — tidak bisa diakses atau diubah langsung dari luar class.

👁 Getter

Getter untuk semua field: username, password, email — membaca nilai private secara terkontrol.

✅ Validasi Setter

Username ≥ 5 karakter • Password ≥ 8 karakter • Email harus mengandung @.

💡Jika validasi gagal, setter melempar Exception. Gunakan try-catch di main() untuk menangkap error dan memastikan program tidak berhenti saat ada data tidak valid.
dart
class User {
  // ── Field Private ──────────────────────────────────────────
  String _username;
  String _password;
  String _email;

  // ── Constructor ─────────────────────────────────────────
  User(this._username, this._password, this._email);

  // ── Getter untuk semua field ───────────────────────────
  String get username => _username;
  String get password => _password;
  String get email    => _email;

  // ── Setter username — minimal 5 karakter ──────────────────
  set username(String val) {
    if (val.length < 5) {
      throw Exception('Username minimal 5 karakter!');
    }
    _username = val;
    print('[OK] Username diubah: $_username');
  }

  // ── Setter password — minimal 8 karakter ───────────────────
  set password(String val) {
    if (val.length < 8) {
      throw Exception('Password minimal 8 karakter!');
    }
    _password = val;
    print('[OK] Password berhasil diubah.');
  }

  // ── Setter email — harus mengandung '@' ─────────────────────
  set email(String val) {
    if (!val.contains('@')) {
      throw Exception('Email harus mengandung "@"!');
    }
    _email = val;
    print('[OK] Email diubah: $_email');
  }

  // ── Method tampilkan() ────────────────────────────────────
  void tampilkan() {
    print('================================');
    print('  Username : $_username');
    print('  Password : ${"*" * _password.length}'); // tampil sebagai bintang
    print('  Email    : $_email');
    print('================================');
  }
}

void main() {
  // Buat objek User dengan data awal
  var user = User('rifqi1', 'password1', 'rifqi@mail.com');
  user.tampilkan();

  // ── Modifikasi dengan nilai VALID ────────────────────────
  user.username = 'prayoga';           // ✓ 7 karakter — valid
  user.password = 'newpass99';         // ✓ 9 karakter — valid
  user.email    = 'prayoga@gmail.com'; // ✓ ada '@' — valid
  user.tampilkan();

  // ── Modifikasi dengan nilai TIDAK VALID ──────────────────
  try {
    user.username = 'rif';        // ✘ hanya 3 karakter
  } catch (e) { print('[ERROR] $e'); }

  try {
    user.password = 'abc';        // ✘ hanya 3 karakter
  } catch (e) { print('[ERROR] $e'); }

  try {
    user.email = 'emailtanpatanda'; // ✘ tidak ada '@'
  } catch (e) { print('[ERROR] $e'); }

  // Data tidak berubah karena semua setter di atas gagal
  user.tampilkan();
}
